# 设计项目云端持久化存储方案

非常抱歉，您之前的项目丢失是因为当前系统处于**演示模式**，数据仅保存在浏览器的临时内存中，刷新页面或重启后会自动重置。

为了永久保存您的设计成果，我将为您实现**云端持久化存储**功能。

## 1. 数据库升级 (Supabase)
我们将创建一个新的数据表 `design_projects`，用于专门存储您的设计作品。
*   **表结构**：
    *   `id`: 项目唯一标识。
    *   `user_id`: 关联您的账户，确保数据私密性。
    *   `type`: 项目类型（如 `manual` 招商手册）。
    *   `title`: 项目名称。
    *   `content`: 完整的 JSON 数据（包含所有幻灯片内容、配图描述等）。
    *   `created_at` / `updated_at`: 时间戳。

## 2. 后端接口开发 (Backend)
在 `main.py` 中新增一组标准的管理接口：
*   **保存/新建** (`POST /api/design/projects`)：将前端编辑器中的数据保存到数据库。
*   **获取列表** (`GET /api/design/projects`)：进入页面时加载您名下的所有项目。
*   **删除项目** (`DELETE /api/design/projects/{id}`)：永久删除指定项目。
*   **获取详情** (`GET /api/design/projects/{id}`)：点击卡片时加载完整内容以供编辑。

## 3. 前端对接 (Frontend)
升级 `FeaturePage.tsx` 的数据逻辑：
*   **初始化加载**：页面加载时自动从后端获取真实的项目列表，替换当前的测试数据。
*   **自动/手动保存**：在 AI 生成完成或您点击“保存”按钮时，将数据同步到云端。
*   **鉴权集成**：确保所有请求都携带您的登录凭证（Token），保证数据归属正确。

## 4. 预期效果
实施完成后，您设计的“招商手册”将永久保存在系统中。无论您何时刷新页面或重新登录，都能在列表页找到它，并点击继续编辑。
